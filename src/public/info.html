SECI DPS Dashboard - User Guide

← Back to Home

This guide will help you understand and use the SECI DPS Dashboard. It explains how Departments, Statistics, Entities, and Fields work together, and how to add or edit them through the dashboard interface.

1. Departments

Departments represent the various business verticals or internal divisions within SECI. Each department can manage its own data independently.

  • Example: Business Development (BD), Project Management (PMC), Contract & Procurement (C&P)
  • Admins can enable/disable departments for visibility on the dashboard.

2. Statistics

Each department can define multiple statistics — these are the high-level categories under which information is grouped.

  • Example: Installed Capacity, Project Pipeline, Financial Performance
  • A statistic must be created before any entities can be added under it.

To Add a Statistic:

  1. Go to the Department dashboard.
  2. Click “Add Statistic”.
  3. Enter a title and optional description.
  4. Click Save.

3. Entities

Entities represent specific units or categories of data within a statistic. Think of them as rows under a heading.

  • Example under “Installed Capacity”: Solar, Wind, Hybrid
  • Each entity must belong to a department + statistic combination.

To Add an Entity:

  1. Navigate to a specific Statistic inside your Department.
  2. Click “Add Entity”.
  3. Enter the entity name and an initial value (if applicable).
  4. Click Save.

4. Entity Fields

Fields are sub-parameters or metadata associated with an entity. These help in breaking down the data further.

  • Example: For entity “Solar”, fields could be Commissioned, Under Construction
  • Each field stores a label and a value.

To Add a Field:

  1. Under the required Entity, click “Add Field”.
  2. Enter a label (e.g., “Commissioned Capacity”) and its value.
  3. Click Save.

5. Editing Existing Data

You can edit existing departments, statistics, entities, and fields using the respective edit buttons:

  • Statistics: Edit name or description.
  • Entities: Edit name or update value.
  • Fields: Edit label and value.

Changes reflect immediately and are stored securely.

6. Headcount and Correspondence

Some entities may also include department-specific metadata like headcount or official correspondence tracking.

  • Admins can update headcount through the Edit button.
  • Documents and letters can be uploaded for correspondence tracking.

7. User Access Control

Admins can manage access for each user:

  • Enable or disable users.
  • Grant department-specific edit permissions.

Each user will only be able to view and modify the departments they’ve been assigned to.

8. Tips and Best Practices

  • Always check for duplicate entity names under the same statistic.
  • Use meaningful field labels and avoid abbreviations unless standardized.
  • Keep statistics updated monthly or quarterly as required.
